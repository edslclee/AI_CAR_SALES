# PRD 생성 규칙

## 목적
- AI가 결정론적으로 제품 요구사항 문서(PRD)를 생성하도록 한다.

## 사전 질문
1. 대상 시장/국가, 언어/통화/단위
2. 초기 출시 범위(Web/Mobile)
3. 추천 기준 가중치
4. 데이터 확보 경로
5. 로그인 방식
6. 개인정보 보관 정책

## PRD 구성
1. 개요/비전/목표
2. 페르소나 & 시나리오
3. 범위(포함/제외)
4. UX 개요
5. API 설계
6. 데이터 모델
7. 아키텍처
8. 비기능 요구사항
9. 테스트 전략
10. 릴리즈 계획
